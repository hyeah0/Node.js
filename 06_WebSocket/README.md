# 웹소켓이란?

- 양방향 데이터 전송을 위한 기술
- 웹서버가 지속적으로 연결된 라인을 통해 실시간으로 데이터를 주고 받을 수 있다.
- WS 프로토콜 사용
- HTML5에 새로 추가된 스펙

- 노드에서는 <b>WS</b>나 <b>Socket.IO</b> 패키지를 통해 웹 소켓 사용 가능
- 이벤트 기반으로 작동

# 폴링(Polling)

- 웹소켓 나오기 전 HTTP 기술을 사용한 실시간 데이터 전송
- 클라이언트에서 서버로 향하는 단방향통신

# 서버센트 이벤트(Server Sent Events => SSE)

- Event Source 객체를 사용한다.
- 한번만 연결하면 서버가 클라이언트로 지속데이터를 보낸다.
- <b>서버에서 클라이언트로 데이터</b>를 보낸다
  - 클라이언트에서 서버로 데이터를 보낼수는 없다.
- 단방향 통신
